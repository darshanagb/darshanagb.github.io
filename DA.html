1.1 Introduction to Data Analytics

What is Data Analytics?

Data Analytics is the process of examining datasets to draw conclusions about the information they contain. It involves systematic computational analysis of data to discover, interpret, and communicate meaningful patterns.

Four Types of Data Analytics:

Type Question Description
Descriptive What happened? Summarizing historical data, understanding patterns and trends
Diagnostic Why did it happen? Finding causes of outcomes, identifying correlations
Predictive What will happen? Forecasting future trends using statistical models
Prescriptive What should we do? Recommending actions and optimization strategies

The Data Analytics Workflow:

  1. Define the problem - Understand business objectives
  2. Collect data - Gather relevant information
  3. Clean and prepare data - Handle missing values, outliers
  4. Explore and analyze - Discover patterns and insights
  5. Visualize findings - Create compelling charts and graphs
  6. Communicate insights - Present actionable recommendations

1.3 Statistical Concepts

Measures of Central Tendency

Understanding Central Tendency

  • Mean (Average): Sum of values Ã· Number of values
    • Sensitive to outliers
  • Median (Middle value): The middle value when data is sorted
    • Robust to outliers
  • Mode: Most frequently occurring value
    • Can have multiple modes

Example 3: Central Tendency

salaries = np.array([45000, 50000, 52000, 55000, 58000, 60000, 65000, 120000]) mean_salary = salaries.mean() median_salary = np.median(salaries) print(f"Mean salary: ${mean_salary:,.2f}") print(f"Median salary: ${median_salary:,.2f}")
Mean salary: $63,125.00 Median salary: $56,500.00 Notice: Mean is higher due to the outlier (120000)

Measures of Dispersion

Understanding Spread of Data

  • Range: Maximum - Minimum
  • Variance: Average squared deviation from mean
  • Standard Deviation: Square root of variance
  • Interquartile Range (IQR): Q3 - Q1 (middle 50%)

Example 4: Dispersion Measures

data = np.array([10, 12, 13, 14, 15, 16, 17, 18, 20]) print(f"Range: {data.max() - data.min()}") print(f"Variance: {data.var():.2f}") print(f"Standard Deviation: {data.std():.2f}") Q1 = np.percentile(data, 25) Q3 = np.percentile(data, 75) print(f"IQR: {Q3 - Q1}")
Range: 10 Variance: 8.89 Standard Deviation: 2.98 IQR: 5.0

HOUR 3: Exploratory Data Analysis (EDA)

3.1 Univariate Analysis

What is Univariate Analysis?

Examining ONE variable at a time to understand its distribution, central tendency, and spread.

For NUMERICAL variables:

  • Distribution (histogram, density plot)
  • Central tendency (mean, median)
  • Spread (std, range, quartiles)
  • Outliers (box plot)

For CATEGORICAL variables:

  • Frequency counts
  • Proportions
  • Bar charts and pie charts

Example 12: Analyzing Numerical Variables

# Statistical summary of Price print(df['Price'].describe()) # Create visualizations import matplotlib.pyplot as plt # Histogram plt.figure(figsize=(10, 6)) plt.hist(df['Price'], bins=20, edgecolor='black') plt.title('Price Distribution') plt.xlabel('Price ($)') plt.ylabel('Frequency') plt.show() # Box plot for outlier detection plt.boxplot(df['Price']) plt.title('Price Box Plot') plt.ylabel('Price ($)') plt.show()

Example 13: Analyzing Categorical Variables

# Frequency counts product_counts = df['Product'].value_counts() print(product_counts) # Proportions product_proportions = df['Product'].value_counts(normalize=True) print(product_proportions) # Bar chart product_counts.plot(kind='bar', color='skyblue') plt.title('Product Sales Frequency') plt.xlabel('Product') plt.ylabel('Count') plt.show()

3.2 Bivariate Analysis

What is Bivariate Analysis?

Examining relationships between TWO variables:

1. Numerical vs Numerical:

  • Scatter plots
  • Correlation coefficients
  • Line plots

2. Numerical vs Categorical:

  • Box plots by group
  • Bar charts with statistics

3. Categorical vs Categorical:

  • Cross-tabulation
  • Stacked bar charts

Example 14: Numerical vs Numerical

# Calculate correlation correlation = df[['Price', 'Quantity', 'Revenue']].corr() print(correlation) # Scatter plot plt.scatter(df['Price'], df['Revenue'], alpha=0.5) plt.title('Price vs Revenue') plt.xlabel('Price ($)') plt.ylabel('Revenue ($)') plt.show() # Correlation heatmap import seaborn as sns sns.heatmap(correlation, annot=True, cmap='coolwarm') plt.title('Correlation Heatmap') plt.show()
ðŸ’¡ Correlation Interpretation:
  • 1.0: Perfect positive correlation
  • 0.7 to 0.9: Strong positive correlation
  • 0.4 to 0.6: Moderate correlation
  • 0.0: No correlation
  • -1.0: Perfect negative correlation

3.3 Time Series Analysis

Time Series Components

Analyzing data over time to identify:

  • Trends: Long-term increase or decrease
  • Seasonality: Regular patterns that repeat
  • Cycles: Periodic fluctuations
  • Irregular variations: Random fluctuations

Example 17: Time Series Analysis

# Aggregate revenue by date daily_revenue = df.groupby('Date')['Revenue'].sum() # Calculate rolling statistics (moving average) rolling_mean = daily_revenue.rolling(window=7).mean() rolling_std = daily_revenue.rolling(window=7).std() # Plot time series plt.figure(figsize=(12, 6)) plt.plot(daily_revenue, label='Daily Revenue', alpha=0.5) plt.plot(rolling_mean, label='7-Day Moving Average', linewidth=2, color='red') plt.title('Daily Revenue Over Time') plt.xlabel('Date') plt.ylabel('Revenue ($)') plt.legend() plt.show() # Monthly aggregation df['YearMonth'] = df['Date'].dt.to_period('M') monthly_revenue = df.groupby('YearMonth')['Revenue'].sum() monthly_revenue.plot(kind='bar', color='green') plt.title('Monthly Revenue') plt.ylabel('Revenue ($)') plt.show()

3.4 Advanced EDA Techniques

Outlier Detection

Outlier Detection Methods

1. IQR Method (Interquartile Range):

  • Lower bound: Q1 - 1.5 Ã— IQR
  • Upper bound: Q3 + 1.5 Ã— IQR
  • Values outside these bounds are outliers

2. Z-Score Method:

  • Calculate: z = (x - Î¼) / Ïƒ
  • Values with |z-score| > 3 are outliers

3. Visual Inspection:

  • Box plots
  • Scatter plots

Example 18: Outlier Detection Using IQR

# Calculate quartiles and IQR Q1 = df['Revenue'].quantile(0.25) Q3 = df['Revenue'].quantile(0.75) IQR = Q3 - Q1 # Define bounds lower_bound = Q1 - 1.5 * IQR upper_bound = Q3 + 1.5 * IQR # Identify outliers outliers = df[(df['Revenue'] < lower_bound) | (df['Revenue'] > upper_bound)] print(f"Q1: ${Q1:.2f}") print(f"Q3: ${Q3:.2f}") print(f"IQR: ${IQR:.2f}") print(f"Lower bound: ${lower_bound:.2f}") print(f"Upper bound: ${upper_bound:.2f}") print(f"Number of outliers: {len(outliers)}") print(f"Outlier percentage: {len(outliers)/len(df)*100:.2f}%")

Example 19: Comprehensive EDA Summary

# Generate summary statistics summary = { 'Total Records': len(df), 'Total Revenue': f"${df['Revenue'].sum():,.2f}", 'Average Order Value': f"${df['Revenue'].mean():,.2f}", 'Most Popular Product': df['Product'].mode()[0], 'Average Rating': f"{df['Rating'].mean():.2f}", 'Date Range': f"{df['Date'].min()} to {df['Date'].max()}" } for key, value in summary.items(): print(f"{key}: {value}")

Best Practices for Data Analytics

1. Always Start with Data Understanding

  • Inspect structure, types, and quality
  • Check for missing values and duplicates
  • Understand the business context

2. Clean Before You Analyze

  • Handle missing values appropriately
  • Remove or fix duplicates
  • Ensure correct data types
  • Document your cleaning steps

3. Create Meaningful Features

  • Derive new variables from existing ones
  • Use domain knowledge
  • Create categories and bins when appropriate

4. Visualize, Visualize, Visualize

  • Pictures tell better stories than numbers
  • Use appropriate chart types for your data
  • Make visualizations clear and interpretable
  • Include titles, labels, and legends

5. Look for Patterns and Relationships

  • Start with univariate analysis
  • Move to bivariate relationships
  • Consider multivariate interactions
  • Think about time dependencies

6. Communicate Insights Clearly

  • Provide summary statistics
  • Highlight key findings
  • Make actionable recommendations
  • Use visualizations to support your points

Common Data Analytics Pitfalls to Avoid

Pitfall How to Avoid
Not checking for missing data Always run .isnull().sum() first
Ignoring outliers Use box plots and IQR method to detect
Confusing correlation with causation Remember: correlation â‰  causation
Using wrong chart types Histograms for distributions, scatter for relationships

Over-relying on mean Check median and mode too; mean is sensitive to outliers
Forgetting data types Ensure dates are datetime, categories are categorical
